# PRD Template

Use this template when the user asks for a full PRD. Keep short PRDs short and omit sections that are not applicable.

```md
# PRD: <product-or-feature>

## 1. Executive Summary
- Product overview
- Core value proposition
- MVP goal statement

## 2. Mission
## 3. Target Users
## 4. MVP Scope
### In Scope
- ✅ ...
### Out of Scope
- ❌ ...
## 5. User Stories
## 6. Functional Requirements
## 7. Non-Functional Requirements
## 8. Architecture & Patterns
## 9. Technology Stack
## 10. Security & Configuration
## 11. API Specification
## 12. Success Criteria
## 13. Implementation Phases
## 14. Risks & Mitigations
## 15. Open Questions
## 16. Appendix
```

## Quality Criteria

- [ ] Executive summary is concise.
- [ ] MVP scope is explicit.
- [ ] Out-of-scope items are explicit.
- [ ] User stories include benefits.
- [ ] Requirements are testable.
- [ ] Success criteria are measurable.
- [ ] Assumptions are labeled.
- [ ] Open questions are visible.
